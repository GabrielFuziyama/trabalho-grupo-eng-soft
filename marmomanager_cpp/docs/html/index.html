Visão Geral do Sistema MarmoManager

O MarmoManager foi reescrito em C++ sob rigorosos Padrões de Projeto e separação em camadas (MVC). Como o sistema gráfico C++ não possui dependências nativas como o Graphviz, nós preparamos abaixo um diagrama fluido, vetorial e interativo desenhado via Mermaid.js, que detalha perfeitamente a Hierarquia de Classes e os Padrões Arquiteturais:

Componentização

Toda a lógica de Controllers e Repositories (Model) é compilada como uma biblioteca dinâmica (libmarmocore.so). As Views (MarmoManagerApp e StatusHistoryDialog) são empacotadas no executável final e apenas consomem a biblioteca dinâmica.
